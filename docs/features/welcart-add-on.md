# Welcart Add-on（拡張）設計案

## 1. 目的

- `lineconnect` 本体は「WPユーザー連携」を提供し続けつつ、Welcart 連携機能は **別プラグイン（拡張）** として分離する。
- これにより、Welcart を利用しないユーザーには余計な依存が発生せず、Welcart 対応機能を有料/別配布にしやすくなる。

## 2. 前提（調査結果）

- Welcart は **独自 DB で会員情報（`usces_member` 等）を管理している**。
- そのため、Welcart 会員を LINE 送信対象とする場合は、**Welcart 側の会員情報を LINE Connect の送信対象に変換する仕組みが必要**。

## 3. 期待する構成（拡張プラグインの設計）

### 3.1 拡張プラグインとして実装する理由
- 本体 (`lineconnect`) に Welcart 固有のコードを混ぜずに済む
- Welcart 連携が必要なユーザーだけが追加で導入できる
- 販売/ライセンス構造（本体無料 + Welcart 有料など）を取りやすい

### 3.2 連携の基本方針
1. **`lineconnect` 本体は「送信対象を決めるためのフック/フィルター」を公開**する。
2. **拡張プラグイン側で Welcart の会員/注文イベントにフックし、LINE 送信処理へ紐付ける。**
3. Welcart 会員が WP ユーザーとして存在する場合は、**既存の WP ユーザー連携ロジックを再利用**する。

### 3.3 連携ポイント例（本体側に用意する想定の拡張ポイント）
- `LineConnect::FILTER_PREFIX . 'user_to_line'`（ユーザー ID → LINE 連携情報変換）
- `LineConnect::FILTER_PREFIX . 'send_message'`（送信処理のカスタマイズ／差し替え）

> ※上記はあくまで設計例。具体的なフック名は本体の実装状況に合わせて決める。

## 4. Welcart 会員情報の扱い

### 4.1 WP ユーザーと紐付けできる場合
- Welcart 会員が WP ユーザーとして存在する場合は、WP ユーザー ID を元に既存の `lineconnect` の送信関数を使う。
- 例: `lineconnect_send_message_to_user( $wp_user_id, $template )` のように呼び出せるようにする。

### 4.2 Welcart 独自会員のみ（WP ユーザーと紐付かないケース）
- Welcart の独自 DB から会員情報を取得し、**LINE 連携 ID（または WP 側で管理する LINE 連携メタ）へ紐付け**する仕組みを拡張側で実装する。
- 例: `send_welcart_user( $welcart_member_id )` のように、Welcart 会員 ID を受け取って LINE 送信を行うラッパー関数を拡張プラグインに用意する。

## 5. MVP で対応すべきイベント（優先度高）

1. **注文完了通知（購入完了）**
   - 注文番号・金額・商品内容・発送予定日などを LINE 通知
   - Welcart の「注文完了」フックに紐付けて送信

2. **発送通知（出荷完了 / 配送手配完了）**
   - 発送ステータス変更時に LINE 通知
   - Welcart の発送関連フック（例：配送状態変更のアクション）に紐付け

3. **注文ステータス変更通知（キャンセル・返品・保留など）**
   - 注文ステータスが変更されたタイミングで LINE 通知
   - ユーザーが注文状況を追いやすくする

## 6. 次のステップ（実装フェーズへの橋渡し）

1. `lineconnect` 本体の拡張ポイント（フック/フィルター）を洗い出して整理する
2. Welcart 側の主要なアクションフック（注文完了、発送、ステータス変更）と引数を調査・リスト化する
3. 拡張プラグインの骨格を作成し、Welcart が有効な環境で動作確認を行う
4. MVP イベントごとのテンプレート・設定 UI を検討し、運用上の使いやすさを担保する
